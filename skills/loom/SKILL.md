# /loom — 첫 설정 마법사 (& 재설정)

새 사용자가 Loom을 깐 직후, **AI가 그 사람을 이해하게** 만드는 첫 단계.
끝나면 그 사람에 맞는 행동규칙·답변 스타일이 잡혀 있다. 이미 설정한 사람은 같은 명령으로 **다시 조정**한다.

## 절대 원칙 (어기지 말 것)
- **추측 금지.** 성격·취향을 멋대로 단정하지 않는다. *명시적으로 말한 것* 또는 *직접 답한 것*만 쓴다.
- **승인 게이트.** 모든 초안은 제안일 뿐. 사용자가 승인해야 파일에 쓴다. 조용히 고치지 않는다.
- **프라이버시 우선.** 사용자가 켠 범위만 본다. **전체 디스크 스캔 금지.**

## 절차

### 0단계 — 이미 설정돼 있나? (재실행 분기)
먼저 `CLAUDE.md` / `output-styles/`에 이전 `/loom` 결과(개인 규칙·스타일)가 이미 있는지 본다.
- **있으면** `ask_user`로 묻는다: **새로 시작**(기존 갈아엎기) / **수정**(빈 곳·바꿀 것만) / **취소**.
  - 수정 선택 시 → 기존 규칙을 보여주고, 바꿀 항목만 골라 해당 단계로.
- **처음이면** 바로 1단계.

### 1단계 — 범위 동의 (복수선택)
`AskUserQuestion`(multiSelect)로 "당신을 파악할 때 어디까지 봐도 될까요? (켠 것만 봅니다)"를 묻는다. 체크박스:
- **☐ 과거 대화 기록** (추천) — `~/.claude/projects`의 지난 대화.
- **☐ 작업했던 프로젝트 폴더** — 아래 *자동 후보*에서 고른 폴더만.
- **☐ 직접 지정한 폴더** — 사용자가 경로를 직접 입력.
- → **아무것도 안 고르면** = 기록 0, 인터뷰만(2단계 인터뷰로 직행).

**작업 폴더 자동 후보 뽑는 법** (전체 디스크 안 뒤짐):
`~/.claude/projects` 폴더명에 *사용자가 과거 작업한 경로*가 인코딩돼 있다. 그걸 디코드해 후보로 제시한다.
- 폴더명 규칙: `C--Foo-Bar` → `C:\Foo\Bar` (맨 앞 `드라이브--`, 나머지 `-`는 `\`).
- `~/.claude` 자신·시스템 폴더(`Windows-System32` 등)·홈 루트는 후보에서 뺀다.
- 후보를 보여주고 사용자가 볼 것만 고르게 한다. **추측해서 임의로 읽지 않는다.**

### 2단계 — 단서 추출 (1단계에서 켠 범위만)
단서 출처가 둘로 나뉜다. extract.py는 **과거 대화 기록만** 읽는다(작업 폴더·지정 폴더는 못 본다). 그래서 폴더는 Claude가 직접 읽어 더한다.

**(A) 과거 대화 기록을 켰으면 → extract.py 실행**
`python3 skills/loom/extract.py --min-count 1`
- 출력 JSON의 `candidates` = 사용자가 직접 친 지시 후보(시스템·도구·훅 텍스트는 걸러짐, 민감정보 가려짐).
- ⚠️ `--min-count 1`로 **넓게** 뽑는다: 진짜 지시는 같은 뜻이라도 매번 다른 문장이라 횟수로 안 뭉친다. 횟수 필터에 의존하지 말고, **3단계에서 Claude가 의미가 비슷한 후보들을 묶는다**(예: "요점만"·"가독성"·"짧게" → "짧게 답하기 선호" 하나로).
- `ok:false`(기록 디렉토리 없음)면 → 이 출처는 비었다고 보고, 폴더도 안 켰으면 3단계 인터뷰로 직행.

**(B) 작업 폴더 / 직접 지정한 폴더를 켰으면 → Claude가 직접 Read**
extract.py가 아니라 **Claude가 그 폴더의 규칙성 문서를 직접 Read**해서 단서에 더한다(README·CLAUDE.md·메모·컨벤션 문서 등 "어떻게 해달라"가 적힌 것). Glob으로 후보를 찾고 해당 파일만 Read한다.
- 대용량·바이너리·빌드 산출물·의존성 폴더(node_modules 등)는 건너뛴다.
- 사용자가 1단계에서 고른 폴더만 본다. 그 밖으로 새어 나가지 않는다.

⚠️ (A)·(B) 모두 단서일 뿐이다. 그대로 규칙화하지 말고 3단계에서 사람에게 확인받는다.

### 3단계 — 확인 + 빈 곳만 인터뷰
- 추출 후보가 있으면: "이런 걸 여러 번 말씀하셨네요 — 맞나요?"로 묶어 제시. 틀린 건 빼고 더할 건 받는다.
- 빠진 핵심은 직접 묻는다 (짧게, 한 번에 하나씩):
  - **답변 스타일**: 자세히 / 요점·가독성 / 짧게 요약 중 어느 쪽?
  - **말투·격식**: 격식 있게(존댓말 등) / 편하게(반말 등) — 어느 쪽? *(쓰는 언어가 한국어가 아니면 그 언어의 격식 기준으로 묻는다. "존댓말/반말"은 예시일 뿐.)*
  - **호칭**: 뭐라고 부를까?
  - (선택) **하는 일·목표** 한 줄.

### 4단계 — 초안 → 승인 → 반영
- 모은 걸 **초안**으로 보여준다: 행동규칙 몇 개 + 답변 스타일.
- 승인받으면 반영한다:
  - 답변 스타일 → `output-styles/`의 스타일 파일(또는 새 output-style 생성)
  - 행동규칙·호칭·목표 → `CLAUDE.md`의 해당 자리
- **승인 없이는 한 줄도 쓰지 않는다.**

### 5단계 — 마무리 안내
"이제 [말투]로 [스타일] 답하겠습니다. 바꾸고 싶으면 언제든 **'앞으로 이렇게 말해'** 라고 하거나 **`/loom`** 을 다시 부르면 됩니다."
→ 이 한 줄이 사용자가 *앞으로 계속 조정하는 법*을 알려준다. (이게 Loom을 살아있게 쓰는 핵심)
