# LOOM — 정본 (단일 진실원천)

> Loom = 사용자가 규칙·피드백을 더할수록 더 personalize되는 개인 시스템(자동 학습 아님 — 사용자 주도+승인). 이 파일 1개가 엔진 설계 정본.
> 설계 원칙은 여기 모은다. 흩어진 옛 설계 문서는 `past/`로 보내고 이 파일이 대체한다.

## 0. 변하지 않는 선 (먼저 읽기)
- **본업이 따로 있고 Loom은 그걸 돕는 부속이다.** (예: 어떤 제품·프로젝트가 본업이고 Loom은 보조 시스템.) 단 Loom 재구축 스프린트 동안만 한시적으로 우선순위를 올릴 수 있다 — 끝나면 본업 우선으로 복귀.
- **모든 자동 제안은 사용자 승인 게이트.** AI 자동적용 절대 금지(아부·자기수정 방어).
- **메타잠식 경계** = 대표적 실패패턴(보조 인프라를 짓다 본업이 죽음). 첫 버전은 일부러 "멍청하고 수동적"으로. 자동판정 욕심 = 함정.

## 1. 구조 = 서랍 5칸 (종류 구분, 개수 아님)
1번만 매 입력 주입. 2~5번은 lint가 질문 키워드에 걸릴 때만 꺼냄.
1. **main (항상 주입)** = 행동 하드라인(예외 거의 없는 것) + "관련 문서 봐라" 라우팅. 작아야 함. (예: `~/.claude/CLAUDE.md`의 핵심 블록 + `ME_CORE.md`)
   - 하드라인: ①사용자 결정 대신 안 내림 ②아부·가짜칭찬 금지 ③생각 먼저·모호하면 멈추고 확인 ④과장 금지(결과로 증명) ⑤곁가지 2단계 이탈 시 멈추고 물음.
   - **가독성·상세수준·도메인규약·수치는 전역 아님** = 기본값일 뿐, 해당 문서가 덮어씀.
2. **너에 대한 지식(보석)** = ABOUT_ME + 피드백 메모(memory) + 인생 슬롯(주제별 최신값, 아래 §2-해법2).
3. **주제 카드** = 도메인별 카드(비전/현황/설계/폴리싱). 수백 개 OK. (예: `cards/`)
4. **엔진 정본** = 이 파일 1개.
5. **핸드오프 NOW** = `NOW.md`, 40줄 상한, 날짜, 휘발(작업일지 금지).

## 2. 마이닝으로 확인된 문제 4 + 해법 (사용자 동의 + 교차검증)
- **문제1 행동강제 안됨**: 규칙 주입은 되나 행동이 안 따라옴. → 해법 A(전역규칙 5개로 축소 + 위반 시 즉시 더 날카롭게 박기) + B(객관적 하드라인 위반만 좁게 사후검사, 비차단 알림부터, 자연어 품질판정 금지). 가독성은 "이 답 맥락문서가 상세 허용했나" 먼저 봄.
- **문제2 상태 노화("자신있게 틀린 답")**: 정보 2분류. ①실물 있는것(코드/파일)=실물 확인. ②인생/말(실물 없음)=주제별 "최신값 1칸" + previous 1칸 보존(current/previous/updated_at/source). **자동 덮어쓰기 금지 → 후보 생성 + 승인.** 누적 X(양 폭발=회수붕괴).
- **문제3 자가발전 큐 정체**: 풀(보러가기) 폐기 → 푸시(아이디어가 사용자에게 옴). **대화 중엔 안 물음, 대화 끝/공백기에 모아서 1개씩 제안→승인.** 명확한 건 조용히 기록, 애매한 것만 끝에 확인. 옛 REVIEW_QUEUE/idle-review OFF.
- **문제4 문서 파편화**: §1 서랍 5칸 + 옛 엔진문서 `past/`로 + 엔진정본 1개(이 파일).
- (문제5 세션시작 맥락부재 = 닫음. lint 매턴 재실행 + "모호하면 되묻기"로 이미 커버.)

## 3. 빌드 순서 (한 번에 X, 단계마다 사용자 검증)
1. **구조 정리**: 옛 문서 past/ + 엔진정본(이 파일) + main 슬림화 + NOW 40줄. 검증=다음 세션에 엉뚱한 옛 문서 안 찾나.
2. **인생 슬롯**: `LIFE_SLOTS.md`(슬롯 정본·4필드·규칙) + ME_CORE 현재값 다이제스트(회수 보장 — ABOUT_ME는 조건부라 항상 읽히지 않음). 후보+승인만. 검증=인생사실 틀리는 빈도 ↓.
3. **푸시 제안**: 세션 끝 최대 1개 제안, 옛 큐 OFF, **외부 봇(디스코드 등)은 처음엔 금지**(대화 중/끝만). 검증=실제 승인/거절 하나.
   - 구현: ME_CORE에 푸시 규칙 1줄(세션 끝·애매한 것 1개·대화중 금지) + 옛 idle-review 스크립트 past/로 이동. **새 큐 파일·봇·훅 안 만듦**(메타잠식 방지 — 푸시는 대화 안에서만, 영속 큐 없음).
   - ⚠️ 운영 환경에 옛 idle-review 스케줄 작업이 남아 있으면 사용자가 직접 해제(관리자 권한 필요할 수 있음).
   - 제안 **출처 2가지**: ①대화에서 발견한 개선점 ②**바깥 기술 스카우팅**(새 AI 기능·스킬·엔지니어링 기법 서칭). ⚠️스카우팅 = 메타잠식 함정(보고서만 쌓고 본업 0보) → 제안 1개씩·승인 필수·본업 시간 잠식 금지. 자동 적용 X.
   - ★규칙 생성 역할분담: **사용자 취향·의도**(AI가 추측 못 하는 것) = 사용자가 직접 말한다. **작업 패턴·반복 실수 교훈·더 나은 방식**(AI가 작업 중 발견하는 것) = AI가 "룸에 넣을까?"로 제안한다. 둘 다 켜둬야 함 — 한쪽만이면 규칙 공백 생김.
   - ★표현 규칙: 문서 만들 제안은 **"룸에 넣을까?"**로 말한다(어디 들어가는지 바로 와닿게). 만든 뒤 어디 사는지도 명시: 주제 지식=`cards/`(트리거 달면 자동 회수) / 인생 사실=LIFE_SLOTS / 행동 규칙=ME_CORE / 로그·기록=일반 파일(트리거 없음).
4. **행동훅 B**: 기존 `hooks/codex-gate.py`가 이미 비차단 객관알림 패턴(코드수정·검토신호·NOW누락) — 거기에 **거울 5슬롯 상한 기계검사 1개만** 추가(`_mirror_slot_overflow`, 줄 세기뿐·NL 판정 0·비차단). 새 훅 안 만듦. 검증=오작동 적나(체감).

## 4. 후보 도구 (3단계에서 비교 — 지금 도입 X = 메타잠식)
- 자율 에이전트·세션 자동기록/압축/주입 류의 기성 메모리 도구들.
- ⚠️ "AI 자동 압축·요약" 류는 우리 "승인식 덮어쓰기" 원칙과 충돌(오요약=기억 썩음). 차용한다면 "승인 UI"·"맥락 주입" 부분만.

## 6. 평가(eval) 로그 — `loom/EVAL.md`
- 본업 작업에 쓰면서 룸의 **회수 적중 / 미스 / 과발동**을 그때그때 한 줄씩 기록. = "잘 된다" 대신 **증거**.
- ⚠️ 카드 아님 — 트리거 없음, 자동 주입 안 됨. 룸을 *바꾸는* 문서가 아니라 *지켜보는* 로그(건강검진부).
- 목적: ①약점 패턴 발견(고칠 거리) ②오픈소스/공유 시 "검증됨" 근거(이게 없으면 "재밌는 개인 설정집"으로 보임).
- ❌ "자동 eval 하네스" 만들기 금지(메타잠식). 사람이 눈에 띌 때 한 줄 적는 것까지만.

## 5. 가드레일 (위반 = 메타잠식 재발 신호, 즉시 중단)
- 평상시: "본업 작업 없는 날 = Loom 개선 금지"(빌드 스프린트 끝난 뒤 켬).
- Loom 손보기 주 30분 상한(스프린트 예외). 세션당 새 메모리 1개↓. 뭐 만들기 전 실물 산출 1개 먼저.
- NO-BUILD(재판정 없이 금지): 벡터DB·지식그래프·자동 대량저장·매 답변 자동검사·자동 WRITE(승인 없는 자기수정).
